# Blueprint: 경찰 근무 편성 웹 애플리케이션

## 1. 개요

이 프로젝트는 경찰 지구대 내 각 팀의 근무 편성 담당자를 위한 웹 애플리케이션입니다. 복잡한 근무 규칙(야간 근무 제외, 고정 대기 등)을 시스템에 반영하여 자동화되고 효율적인 근무표 작성을 지원하는 것을 목표로 합니다.

## 2. 2단계 개발 전략

**1단계 (현재 진행): 공개 인터넷 환경에서 기능 완성**
- **목표:** Firebase를 백엔드로 사용하여, 인터넷만 되면 어디서든 접속하고 사용할 수 있는 완벽한 기능의 웹 애플리케이션을 개발합니다. 이를 통해 실제 작동하는 프로토타입을 확보합니다.
- **기술 스택:** React, Cloudflare(배포), Firebase(인증, 데이터베이스)

**2단계 (추후 계획): 경찰 내부망으로 이전**
- **목표:** 1단계에서 완성된 앱을 경찰 내부망 환경으로 이전하여 최종 배포합니다.
- **기술 스택:** React, 내부망 웹서버(Nginx 등), 자체 구축 API 서버(Node.js 등), 자체 구축 DB(PostgreSQL 등)

---

## 3. 1단계 상세 실행 계획

### 현황 분석 (2023-10-27 기준)

- **장점:** React 컴포넌트 기반의 깔끔한 구조, 풍부한 UI/UX 및 핵심 기능(근무표, 직원관리, 설정 등) 대부분이 구현됨.
- **핵심 개선 과제:** 모든 데이터가 `localStorage`에 저장되어 있어 다중 사용자 지원, 데이터 영속성, 보안이 불가능한 상태.

### 실행 계획: `localStorage`를 `Firebase`로 전환

**1. Firebase 프로젝트 초기화 (완료)**
   - **작업:** `firebase_init` 도구를 사용하여 현재 React 프로젝트에 Firebase SDK를 연동하고 기본 설정을 구성합니다.
   - **결과물:** 프로젝트에 Firebase 연동 환경 구축.

**2. Firebase Authentication (인증) 도입 (완료 및 개선)**
   - **작업:**
     - Firebase 콘솔에서 `이메일/비밀번호` 인증 방식 활성화.
     - 로그인/회원가입 UI 컴포넌트 생성.
     - 앱의 최상위 레벨에서 로그인 상태를 체크하여, 로그인하지 않은 사용자는 로그인 페이지로 리디렉션하는 로직 구현.
     - **최근 수정 (2026-03-16):** 
       - 로그인 오류 메시지 한국어화 (상세 에러 코드 대응).
       - 비밀번호 재설정(Password Reset) 기능 추가.
       - 헤더에 로그아웃 버튼 추가 및 로그인 세션 관리 개선.
       - 회원가입 시 비밀번호 확인(Confirm Password) 필드 추가로 오타 방지.
   - **결과물:** 사용자 로그인 및 접근 제어 기능.

**3. Firebase Firestore (데이터베이스) 설계 및 마이그레이션 (진행중)**
   - **작업:**
     - **데이터 모델링:**
       - `users`: 사용자 계정 정보 (uid, email, teamId 등)
       - `employees`: 전 직원 명단 (기존 `employees` 상태)
       - `rosters`: 날짜별, 팀별 근무표 정보 (기존 `schedules` 데이터)
       - `specialNotes`: 직원별 특이사항 (기존 `specialNotes` 상태)
       - `settings`: 전역 설정 (기존 `settings` 상태)
     - **로직 변경:**
       - `useState`와 `localStorage`를 사용해 상태를 초기화하던 부분을 `useEffect`와 Firestore 데이터 조회 로직으로 전면 교체.
       - **최근 수정 (2026-03-16):**
         - 데이터 로딩 중 초기 데이터(INITIAL_EMPLOYEES)가 잠깐 보이는 현상 방지 및 로딩 상태 처리 강화.
         - 트렌디한 로딩 화면(스피너 및 애니메이션) 도입으로 사용자 경험 개선.
         - 데이터가 없을 경우 '데모 데이터'임을 알리는 인디케이터 추가.
         - 누락되었던 '중점 구역', '시간대' 설정 UI 복구 및 저장 로직 확인.
         - '근무 유형 관리' 카드에 부가 설명 추가.
   - **결과물:** 모든 데이터가 중앙 데이터베이스에서 관리되는 애플리케이션.

**4. Firestore 보안 규칙 적용**
   - **작업:**
     - 사용자가 로그인했는지 여부를 체크하는 기본 규칙 작성 (`request.auth != null`).
     - `schedules`와 `employees` 컬렉션에 대해, 로그인한 사용자의 `teamId`와 문서의 `teamId`가 일치하는 경우에만 읽기/쓰기를 허용하는 규칙을 작성.
     - 관리자급 사용자를 위한 별도 권한 규칙 고려.
   - **결과물:** 데이터 무결성 및 팀별 권한 분리 확보.
